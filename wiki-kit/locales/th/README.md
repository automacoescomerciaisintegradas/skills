# wiki-kit

เทมเพลตฐานความรู้แบบ Markdown ที่ออกแบบมาเพื่อให้ LLM เป็นผู้ดูแลรักษา

แทนที่จะค้นหาวัตถุดิบต้นทางใหม่ทุกครั้งที่มีคำถาม LLM จะสะสมสรุป การอ้างอิงข้าม และการวิเคราะห์ไว้ใน `wiki/` เพื่อสร้างชั้นความรู้ที่คงอยู่ต่อเนื่องตามเวลา

เทมเพลตนี้นำเวิร์กโฟลว์ที่เสนอไว้ใน gist "[LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)" ของ karpathy มาใช้งานจริง

## เริ่มต้นอย่างรวดเร็ว

```bash
npx create-wiki-kit my-wiki
cd my-wiki
```

จากนั้น:

1. เขียนสิ่งที่คุณต้องการค้นคว้าไว้ใน `wiki/overview.md`
2. วางวัสดุต้นทางไว้ใน `raw/sources/`
3. ให้ LLM อ่าน `CLAUDE.md` แล้วทำการ ingest วัสดุเหล่านั้น

## LLM ทำอะไรบ้าง

เมื่ออ้างอิงไปที่ `CLAUDE.md` แล้ว LLM จะทำตามเวิร์กโฟลว์ที่กำหนดไว้:

- **Ingest**: อ่านแหล่งข้อมูลดิบ สร้างสรุปใน `wiki/sources/` อัปเดตหน้าคอนเซปต์และเอนทิตีที่เกี่ยวข้อง แล้วบันทึกงานลงใน `index.md` และ `log.md`
- **Query**: ตรวจสอบ wiki ก่อน ตอบโดยมีหลักฐานรองรับ และบันทึกการวิเคราะห์ที่นำกลับมาใช้ซ้ำได้ไว้ใน `wiki/syntheses/`
- **Lint**: ตรวจสอบ wiki เป็นระยะเพื่อหาความขัดแย้ง เนื้อหาที่ล้าสมัย หน้า orphan และคอนเซปต์สำคัญที่ยังไม่ได้ยกระดับ

## โครงสร้างไดเรกทอรี

หลังจากสร้างโปรเจกต์แล้ว โครงสร้างจะเป็นดังนี้:

```text
my-wiki/
├── CLAUDE.md              # กฎการทำงานสำหรับ LLM
├── AGENTS.md              # ชี้เอเจนต์อื่นไปยัง CLAUDE.md
├── README.md
├── raw/
│   ├── sources/           # บทความ PDF โน้ต ถอดความ ฯลฯ
│   └── assets/            # รูปภาพ ไดอะแกรม ไฟล์แนบ
├── wiki/
│   ├── overview.md        # หัวข้อ วัตถุประสงค์ สมมติฐาน
│   ├── index.md           # ดัชนีตามเนื้อหา
│   ├── log.md             # บันทึกตามลำดับเวลาของทุกการดำเนินการ
│   ├── open-questions.md  # คำถามที่ยังไม่คลี่คลายและหัวข้อวิจัยต่อ
│   ├── sources/           # หน้าสรุปหนึ่งหน้าต่อวัสดุดิบหนึ่งชิ้น
│   ├── concepts/          # ธีมที่เกิดซ้ำ ประเด็นถกเถียง ศัพท์สำคัญ
│   ├── entities/          # บุคคล บริษัท ผลิตภัณฑ์ องค์กร
│   ├── syntheses/         # การเปรียบเทียบ การวิเคราะห์ ข้อสรุปที่ใช้ซ้ำได้
│   └── maintenance/
│       └── lint-reports/  # รายงานการตรวจทานเป็นระยะ
└── templates/             # โครงสร้างหน้าอ้างอิงสำหรับ LLM
```

## Locale

เทมเพลตนี้มาพร้อมแพ็กภาษา 14 ภาษา ตัวเลือก `--locale` ใช้เลือกภาษาสำหรับ `CLAUDE.md`, เทมเพลต, โครง wiki และ README ทั้งหมด ค่าเริ่มต้นคือ `en`
`README.md` ที่ระดับรีโพนี้อธิบายตัวเทมเพลตเอง ส่วนโปรเจกต์ที่สร้างขึ้นควรได้รับ README ของ locale ที่เลือก เช่น `locales/en/README.md` หรือ `locales/ja/README.md`

| Code | Language    | Code | Language    |
|------|-------------|------|-------------|
| `de` | German      | `ko` | Korean      |
| `en` | English     | `pt` | Portuguese  |
| `es` | Spanish     | `ru` | Russian     |
| `fr` | French      | `th` | Thai        |
| `id` | Indonesian  | `tr` | Turkish     |
| `it` | Italian     | `vi` | Vietnamese  |
| `ja` | Japanese    | `zh` | Chinese     |

```bash
npx create-wiki-kit my-wiki --locale ja
```

หากต้องการเพิ่ม locale ใหม่ ให้สร้าง `locales/<code>/` ในรีโพเทมเพลตและใส่ไฟล์แปลทั้ง 22 ไฟล์ให้ครบ

## ตัวอย่างพรอมป์

### การเริ่มต้น

```text
อ่าน CLAUDE.md และทำความเข้าใจเป้าหมายกับกฎการทำงานของ wiki นี้
จากนั้นตรวจสอบ wiki/overview.md และเตรียมคำถามขั้นต่ำที่จำเป็นเพื่อเติมช่องว่าง
```

### Ingest

```text
ตาม CLAUDE.md ให้ประมวลผลวัสดุจาก raw/sources/ ที่ยังไม่ได้ ingest จำนวนหนึ่งรายการ
สร้าง source summary และอัปเดต concepts / entities / overview หากจำเป็น
จากนั้นอัปเดต index.md และ log.md
```

### Query

```text
อ่าน wiki/index.md ก่อน แล้วค่อยดูหน้าที่เกี่ยวข้องใน wiki
สรุปข้อถกเถียงหลัก 3 ข้อของหัวข้อนี้ พร้อมระบุจุดที่หลักฐานยังอ่อน
ถ้าผลลัพธ์มีคุณค่าสำหรับการใช้ซ้ำ ให้บันทึกไว้ใน syntheses
```

### Lint

```text
ตาม CLAUDE.md ให้ lint ทั้ง wiki
ระบุความขัดแย้ง เนื้อหาที่ล้าสมัย หน้า orphan คอนเซปต์สำคัญที่ยังไม่ได้ยกระดับ
และหัวข้อที่ควรวิจัยต่อ จากนั้นสร้างรายงานใน wiki/maintenance/lint-reports/
แล้วอัปเดต index.md และ log.md
```

## หลักการสำคัญ

- `raw/` เป็นแบบอ่านอย่างเดียวสำหรับ LLM มนุษย์เป็นผู้ใส่วัสดุ และ LLM จะไม่แก้ไข
- `wiki/` คือชั้นความรู้ที่ LLM สะสมและขยาย โดยมีสรุปและการอ้างอิงข้ามเก็บอยู่ที่นี่
- `index.md` และ `log.md` ต้องอัปเดตทุกครั้งที่มีการดำเนินการ
- ผลลัพธ์การ query ที่มีคุณค่าสูงควรถูกบันทึกใน `syntheses/` แทนที่จะปล่อยให้หายไปในบทสนทนา
- การ lint เป็นระยะช่วยจับความขัดแย้งและช่องว่างก่อนที่จะสะสมมากขึ้น

## การใช้งานร่วมกับเอเจนต์อื่น

กฎการทำงานถูกกำหนดไว้ใน `CLAUDE.md` สำหรับเอเจนต์ที่อ่าน `AGENTS.md` (เช่น Codex) ไฟล์นั้นจะชี้ไปยัง `CLAUDE.md` หากจำเป็นให้คัดลอกเนื้อหาไปยังรูปแบบการตั้งค่าของเอเจนต์นั้นโดยตรง
