# wiki-kit

Шаблон базы знаний на Markdown, рассчитанный на сопровождение со стороны LLM.

Вместо того чтобы заново искать по сырым материалам при каждом вопросе, LLM накапливает в `wiki/` резюме, перекрёстные ссылки и анализы, формируя со временем устойчивый слой знаний.

Этот шаблон реализует рабочий процесс, предложенный в gist "[LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)" от karpathy.

## Быстрый Старт

```bash
npx create-wiki-kit my-wiki
cd my-wiki
```

Затем:

1. Опишите, что вы хотите исследовать, в `wiki/overview.md`
2. Поместите исходные материалы в `raw/sources/`
3. Попросите LLM прочитать `CLAUDE.md` и выполнить ingest

## Что Делает LLM

После указания на `CLAUDE.md` LLM следует определённому рабочему процессу:

- **Ingest**: Читает сырой источник, создаёт резюме в `wiki/sources/`, обновляет связанные страницы концепций и сущностей и фиксирует работу в `index.md` и `log.md`.
- **Query**: Сначала обращается к wiki, отвечает с опорой на доказательства и сохраняет переиспользуемые анализы в `wiki/syntheses/`.
- **Lint**: Периодически проверяет wiki на противоречия, устаревший контент, сиротские страницы и непродвинутые важные концепции.

## Структура Каталогов

После scaffolding сгенерированный проект выглядит так:

```text
my-wiki/
├── CLAUDE.md              # Правила работы для LLM
├── AGENTS.md              # Направляет другие агенты к CLAUDE.md
├── README.md
├── raw/
│   ├── sources/           # Статьи, PDF, заметки, расшифровки и т.д.
│   └── assets/            # Изображения, диаграммы, вложения
├── wiki/
│   ├── overview.md        # Тема, цель, гипотезы
│   ├── index.md           # Индекс на основе содержания
│   ├── log.md             # Хронологический журнал всех операций
│   ├── open-questions.md  # Нерешённые вопросы и кандидаты для исследования
│   ├── sources/           # Одна страница резюме на каждый сырой материал
│   ├── concepts/          # Повторяющиеся темы, споры, терминология
│   ├── entities/          # Люди, компании, продукты, организации
│   ├── syntheses/         # Сравнения, анализы, переиспользуемые выводы
│   └── maintenance/
│       └── lint-reports/  # Отчёты периодических ревизий
└── templates/             # Справочные структуры страниц для LLM
```

## Локали

Этот шаблон поставляется с 14 языковыми пакетами. Опция `--locale` выбирает язык для `CLAUDE.md`, шаблонов, wiki-scaffolding и всех файлов README. Значение по умолчанию — `en`.
Этот `README.md` в корне репозитория документирует сам шаблон. Сгенерированные проекты должны получать README выбранной локали, например `locales/en/README.md` или `locales/ja/README.md`.

| Code | Language    | Code | Language    |
|------|-------------|------|-------------|
| `de` | German      | `ko` | Korean      |
| `en` | English     | `pt` | Portuguese  |
| `es` | Spanish     | `ru` | Russian     |
| `fr` | French      | `th` | Thai        |
| `id` | Indonesian  | `tr` | Turkish     |
| `it` | Italian     | `vi` | Vietnamese  |
| `ja` | Japanese    | `zh` | Chinese     |

```bash
npx create-wiki-kit my-wiki --locale ja
```

Чтобы добавить новую локаль, создайте `locales/<code>/` в репозитории шаблона и добавьте переведённые версии всех 22 файлов.

## Примеры Промптов

### Инициализация

```text
Прочитай CLAUDE.md и пойми назначение и правила работы этого wiki.
Затем проверь wiki/overview.md и подготовь минимальный набор вопросов, чтобы закрыть пробелы.
```

### Ingest

```text
Следуя CLAUDE.md, обработай один ещё не ingest-ированный материал из raw/sources/.
Создай резюме источника, при необходимости обнови concepts / entities / overview,
затем обнови index.md и log.md.
```

### Query

```text
Сначала прочитай wiki/index.md, затем обратись к связанным страницам в wiki.
Суммируй три основных аргумента по этой теме и отметь, где доказательная база слаба.
Если результат имеет ценность для повторного использования, сохрани его в syntheses.
```

### Lint

```text
Следуя CLAUDE.md, выполни lint для всего wiki.
Определи противоречия, устаревший контент, сиротские страницы, непродвинутые ключевые концепции
и кандидатов для дальнейшего исследования. Создай отчёт в wiki/maintenance/lint-reports/,
затем обнови index.md и log.md.
```

## Основные Принципы

- `raw/` доступен LLM только на чтение; материалы туда помещают люди, а LLM их не изменяет
- `wiki/` — это слой знаний, который развивает LLM; здесь накапливаются резюме и перекрёстные ссылки
- `index.md` и `log.md` обновляются при каждой операции
- Ценные результаты запросов сохраняются в `syntheses/`, а не теряются в разговоре
- Периодические lint-проверки выявляют противоречия и пробелы до того, как они накопятся

## Использование с Другими Агентами

Правила работы определены в `CLAUDE.md`. Для агентов, которые читают `AGENTS.md` (например, Codex), этот файл перенаправляет к `CLAUDE.md`. При необходимости скопируйте содержание в собственный формат конфигурации агента.
